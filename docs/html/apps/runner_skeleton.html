src/apps runner skeleton

Runner as the project skeleton

The runner layer connects Hydra configuration, UnifiedSystemV510 construction, the life loop, IPC control, debug services, visualizers, MuJoCo world creation and training control.

Launch sequence

runner.py
Root compatibility launcher. It preserves the normal command and delegates to the slim Hydra entrypoint.
↓
runner_entry.py
Hydra main(). It prints resolved config, builds the typed runner config, constructs the system through build_unified_system(), then calls system.run().
↓
runner_config.py
Filters unknown top-level/nested OmegaConf keys, merges into UnifiedV510Config, and renders resolved YAML for diagnostics.
↓
runner_system_factory.py
Single construction sequence: instantiate the system, normalize runtime bookkeeping, normalize startup/window/sensor state, ensure services, apply mode.
↓
runner.py / UnifiedSystemV510
Slim compatibility wrapper. It preserves from src.apps.runner import UnifiedSystemV510, assembles mixins and binds main hooks to extracted helpers.
↓
runner_unified_init.py
Actual heavy initialization: device, seeds, teacher/vocab, world model, optimizer, M1, SelfCore, replay, emotional drive, MuJoCo world, dynamic rig, visualizers, checkpoints and services.
↓
runner_loop.py
Runtime life loop: world reset, background train thread, life steps, optional MuJoCo viewer sync, slot-4D streaming, checkpoint save and shutdown.
