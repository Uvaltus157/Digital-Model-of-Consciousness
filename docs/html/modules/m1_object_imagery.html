M1_OBJECT_IMAGERY

Модуль M1: внутреннее объектное воображение

M1 превращает поток сенсоров в устойчивые внутренние object slots, хранит латентный объект z_obj, декодирует его в 2D/3D/4D представления и отдаёт объектную сцену другим модулям.

нейросетевой модуль сенсоры → fusion object slots / latent memory

Назначение

M1 — нижний объектный слой сознательной модели. Его задача: собрать разрозненные сенсорные признаки в компактный внутренний объект, который можно хранить, узнавать, декодировать, сравнивать и использовать как основание для событий, памяти, речи и планирования.

В каноническом реестре модулей M1 описан как цепочка: sensors, fusion, object slots, latent object and 2D/3D/4D reconstruction.

Сон и отключение сенсоров

Runtime M1 содержит центральный sensor-gate. Если video/contact/imu выключены, соответствующие входы зануляются, а скрытые обходные пути вроде geometry visibility не должны притворяться сенсорным восприятием.

Это важно для имитации сна: когда глаза закрыты, M1 должен опираться на внутренний latent и dream dynamics, а не тайно использовать внешнюю сцену.

Минимальная спецификация готовности M1

  • Есть устойчивые z_obj_slots и confidence_slots.
  • Есть активный slot и понятная политика slot binding.
  • Сенсорные отключения не обходятся визуализаторами или geometry state.
  • 2D/3D реконструкция декодируется из внутреннего latent, а не из прямой копии сенсора.
  • Изменение slot-а может быть передано в M2 как событие.
  • Debug UI показывает slots, confidence, dream activation и reconstruction.
