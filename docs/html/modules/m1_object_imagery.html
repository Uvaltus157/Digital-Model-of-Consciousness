M1_OBJECT_IMAGERY

M1: internal object imagery

M1 transforms sensor streams into stable internal object slots, maintains latent object memory, reconstructs objects in 2D/3D/4D and exposes object state to event memory, self core, world model, metacognition and planning.

neural modulesensors → fusionobject slots / memory2D/3D/4D debug

Purpose

M1 is the first internal object layer of the conscious model. It should not merely mirror camera frames. Its job is to create an internal object that keeps identity over time, has confidence, can be decoded, compared and used as a base for events, memory, language and planning.

The main output of M1 is z_obj and the set of z_obj_slots. These object slots become inputs to M2, M4, M5, M9, M12 and M15.

Main data structures

z_obj                         # active latent object
z_obj_slots[B, S, D]           # all object slots
confidence_slots[B, S, 1]      # current sensory confidence
memory_stability_slots[B,S,1]  # persistent stability
active_slot_index              # current object of attention
dream_activation_slots         # sleep/dream activation
slot_age                       # slot age / persistence
slot_4d timeline               # temporal 3DGS/xyz history

Main M1 Python files

Links are local repository links. They work when browsing this file inside the repository checkout.

models/object_inner_imagery_3d.py main M1 model

src/modules/m01_object_imagery/models/object_inner_imagery_3d.py

The central M1 model file. It contains ObjectInnerImagery3DConfig, helper functions, VisualTactileObjectFusion, ObjectSlotMemory, 2D/3D reconstruction heads and the overall inner object representation system.

  • ObjectInnerImagery3DConfig: latent, hidden, image, vision, tactile, body, hand, leg, point, voxel, slot and dream settings.
  • VisualTactileObjectFusion: maps multimodal features to z_update and sensor strength signals.
  • ObjectSlotMemory: maintains object slots, confidence, stability, sticky routing and active slot identity.
  • ObjectImaginationHead2D: decodes latent object state into RGB/depth/mask and attributes.

models/object_imagery_decoder.py 2D semantic decoder

src/modules/m01_object_imagery/models/object_imagery_decoder.py

A dedicated decoder for internal object representations. It accepts object_repr and, when context is enabled, appends workspace, thought and reflection vectors. It then creates a feature grid, upsamples with transposed convolutions and emits several heads.

  • ObjectImageryConfig: object_dim, workspace_dim, thought_dim, reflection_dim, hidden_dim, image size/channels and semantic class counts.
  • ObjectImageryDecoder: MLP encoder → latent grid → upsampling → rgb, alpha, depth.
  • Semantic heads: shape/color/material logits, class ids and object confidence.
  • blended_object_image(): blends RGB with alpha and a background value for visualization.

models/slot_4d_deformation.py 4D deformation model

src/modules/m01_object_imagery/models/slot_4d_deformation.py

The Step 3B neural 4D deformation component. It learns a compact per-slot deformation field over sampled 3D Gaussian points.

  • Slot4DDeformationModel: small MLP mapping [xyz, normalized_time] → delta_xyz.
  • Slot4DDeformationTrainer: keeps one deformation model and optimizer per object slot.
  • train_from_timeline(): trains from the last two timeline frames by predicting xyz(t0)+delta → xyz(t1).
  • Slot4DDeformationMetrics: slot id, target, validity, loss, motion norm, predicted delta norm, sample count and temporal dt.

slot_4d_reconstruction.py 4D timeline

src/modules/m01_object_imagery/slot_4d_reconstruction.py

Step 3A: temporal history over per-slot Gaussian states. It does not learn deformation yet; it captures state(t) so Step 3B can learn the deformation.

  • Slot4DFrame: a snapshot of a slot state: live step, gaussian count, loss, confidence, dynamic norm and sampled xyz.
  • Slot4DTimelineBuffer: per-slot frame queue with motion norm, temporal span and mean delta metrics.
  • Slot4DReconstructor: extracts xyz samples from the Gaussian reconstructor and appends timeline frames.

slot_4d_playback.py 4D playback

src/modules/m01_object_imagery/slot_4d_playback.py

Step 3C: deformation-aware playback preview. It applies the learned deformation to the current Gaussian state and renders a time-phase preview using the existing Gaussian renderer/backend.

  • Slot4DPlaybackRenderer: computes playback phase, applies deformation and calls preview rendering.
  • _DeformedGaussianState: read-only wrapper that replaces xyz but keeps scale, opacity and color.
  • Slot4DPlaybackMetrics: render validity, deformation usage, gaussian count, phase, delta norm, backend and fps.

slot_4d_open3d_export.py Open3D export

src/modules/m01_object_imagery/slot_4d_open3d_export.py

Exports raw and deformed slot point clouds to a compressed .npz file so a separate Open3D viewer can inspect the latest 4D slot state.

  • Slot4DOpen3DExporter: throttled exporter for raw/deformed xyz, color, alpha and metrics.
  • _deformed_xyz(): applies the deformation model for a playback phase when available.
  • export(): writes arrays such as slot_0_raw_xyz, slot_0_deformed_xyz, color, alpha and phase.

runtime.py runtime bridge

src/modules/m01_object_imagery/runtime.py

The main M1 runtime bridge. It connects object imagery with M2/M3/M4/M12/M15 through event latent memory, debug windows, passport memory, action traces, trust gates and scenario runtime mixins.

  • ObjectImageryRuntimeMixin: combines many cross-module runtime mixins used by object imagery.
  • _input_sensor_enabled() and _safe_obs_for_inner_object_viz(): prevent disabled sensors from leaking into visual/debug paths.
  • _ensure_event_latent_memory(): creates the bridge to event sentence memory.
  • update_event_code_visualizer_window(): shows slot vocabulary and event code sentences.

How M1 works during a life step

  1. The runner receives observations from the MuJoCo world: cameras, depth, tactile signals, body state and actions.
  2. M1 runtime creates a safe observation copy respecting sensor gates.
  3. Vision summary and tactile/body/action features are fed into fusion.
  4. Fusion outputs z_update and sensor-strength signals.
  5. ObjectSlotMemory selects a slot and updates only that selected slot.
  6. Decoders produce internal images, shapes, depth, points and voxels.
  7. Runtime updates visualizers and event memory.
  8. 4D subsystem may append a timeline frame, train deformation and export Open3D preview.

How M1 works in sleep

When video/contact/imu are disabled, M1 must not secretly use the external scene. It reads previous slots, can run dream latent dynamics and decodes internal state.

The sleep goal for M1 is not a black frame; it is autonomous internal object replay from saved latent state.

Typical failure modes

  • Slot identity jumping: inspect sticky thresholds and proposal slot lock.
  • Sleep still sees the scene: look for sensor-gate bypass in runtime/visualizers.
  • Confidence always zero: check evidence strength and disabled sensors.
  • 3D output empty: Gaussian state may not exist or confidence may be below threshold.
  • 4D deformation not training: timeline has fewer than two valid frames or an empty sample.

Minimum readiness specification

  • Stable z_obj_slots, confidence_slots and memory_stability_slots.
  • Clear active slot and sticky routing without random switching.
  • Full sensor-gate protection for video/contact/imu.
  • 2D decoding from latent state, not a direct sensor copy.
  • 3D/Open3D output for formed objects.
  • 4D timeline → deformation → playback/export pipeline.
  • Connection to M2 event memory through slot changes.
  • Debug UI shows slot identity, confidence, latent heatmap, reconstruction and sleep masks.
